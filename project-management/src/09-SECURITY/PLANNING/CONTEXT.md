# project-management/src/09-SECURITY/PLANNING

Security gap analysis reports generated by scanning stories, sprints, database schema, and user
flow documents against `project-management/docs/SECURITY-GUIDE.md`.

## Directory Tree

```text
project-management/src/09-SECURITY/PLANNING/
├── CONTEXT.md                          ← this file
├── SECURITY-REPORT-STORIES.md          ← per-story security analysis            [created by planning workflow]
├── SECURITY-REPORT-SPRINTS.md          ← per-sprint security analysis            [created by planning workflow]
├── SECURITY-REPORT-DATABASE.md         ← encryption, RLS, and access control     [created by planning workflow]
└── SECURITY-REPORT-USER-FLOWS.md       ← STRIDE analysis of all user flows       [created by planning workflow]
```

## When to read these

- Before running `workflows/09-security-checks/` on a new sprint
- When assessing the security scope of a story or sprint
- When preparing a threat model or security audit document in `src/09-SECURITY/`
