# project-management/src/12-API-DESIGN/PLANNING

Pre-workflow planning reports generated by scanning stories, sprints, and database schema documents
for missing API design coverage — identifying which user stories still need a GraphQL contract
before they can enter sprint planning.

## Directory Tree

```text
project-management/src/12-API-DESIGN/PLANNING/
├── CONTEXT.md                          ← this file
├── API-REPORT-STORIES.md               ← per-story API design coverage analysis     [created by planning workflow]
├── API-REPORT-SPRINTS.md               ← per-sprint API surface and coverage gaps   [created by planning workflow]
└── API-REPORT-SCHEMA.md                ← per-model API exposure and gap analysis    [created by planning workflow]
```

## When to read these

- Before running `workflows/12-api-design/` on a new story or sprint
- When assessing which stories are blocked from sprint planning due to a missing API design
- When reviewing overall GraphQL surface coverage across the current schema

## Reports

| File                    | Source scanned         | Coverage                                                                 |
| ----------------------- | ---------------------- | ------------------------------------------------------------------------ |
| `API-REPORT-STORIES.md` | `src/01-STORIES/`      | Which stories have a signed-off API design, which are missing one        |
| `API-REPORT-SPRINTS.md` | `src/13-SPRINT-PLANS/` | API design gaps that could block sprint commitment, incomplete contracts |
| `API-REPORT-SCHEMA.md`  | `src/03-DATABASE/`     | Which models have GraphQL types defined, which are unexposed or partial  |

These reports are generated by running the API design planning workflow against the current state
of each source folder. They have not yet been created — run `workflows/12-api-design/` to produce
them.
