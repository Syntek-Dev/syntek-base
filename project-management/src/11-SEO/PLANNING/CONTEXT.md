# project-management/src/11-SEO/PLANNING

SEO gap analysis reports generated by scanning stories, sprints, and user flow documents for
missing metadata, structured data, Core Web Vitals risks, and coverage gaps against
`project-management/docs/SEO-CHECKLIST.md`.

## Directory Tree

```text
project-management/src/11-SEO/PLANNING/
├── CONTEXT.md                      ← this file
├── SEO-REPORT-STORIES.md           ← per-story SEO coverage analysis           [created by planning workflow]
├── SEO-REPORT-SPRINTS.md           ← per-sprint SEO scope and gaps             [created by planning workflow]
└── SEO-REPORT-USER-FLOWS.md        ← per-flow metadata and discoverability gaps [created by planning workflow]
```

## When to read these

- Before running `workflows/11-seo-checks/` on a new sprint
- When assessing the SEO scope of a story or sprint
- When preparing Lighthouse exports or SEO audit files in `src/11-SEO/`

## Reports

| File                       | Source scanned      | Coverage                                                  |
| -------------------------- | ------------------- | --------------------------------------------------------- |
| `SEO-REPORT-STORIES.md`    | `src/01-STORIES/`   | Metadata, OG tags, JSON-LD, canonical URL, slug, sitemap  |
| `SEO-REPORT-SPRINTS.md`    | `src/02-SPRINTS/`   | SEO tasks, blocking gaps, CWV risk, missing AC            |
| `SEO-REPORT-USER-FLOWS.md` | `src/04-USER-FLOW/` | Discoverability, redirects, canonical conflicts, headings |

These reports are generated by running the SEO planning workflow against the current state of each
source folder. They have not yet been created — run `workflows/11-seo-checks/` to produce them.
