# project-management/src/PLANS

Architectural and feature plans for the project-name project.

## Directory Tree

```text
project-management/src/PLANS/
├── CONTEXT.md               ← this file
├── PLAN-<FEATURE-NAME>.md   ← feature-level plans (e.g. PLAN-AUTH.md)
└── PLAN-US###-<DESC>.md     ← story-linked plans (e.g. PLAN-US001-AUTH.md)
```

**Naming:** `PLAN-<FEATURE-NAME>.md` or `PLAN-US###-<DESCRIPTION>.md`

Plans are created before implementation begins. They document the technical
approach, data model, API surface, and open questions.
