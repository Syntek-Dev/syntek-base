# SEO Checks — Steps

**Language**: British English (en_GB)

---

## Steps

### Step 1 — Run the SEO skill

Run the automated SEO skill against the page or pages in scope:

```text
/syntek-dev-suite:seo [describe the page, route, and any specific SEO requirements]
```

The skill checks metadata, Open Graph tags, canonical URLs, JSON-LD, sitemap inclusion,
and heading hierarchy. Review its output before proceeding to manual checks.

### Step 2 — Verify metadata in the browser

Open the page in the browser and inspect the `<head>` tag (DevTools → Elements):

- `<title>` — present, max 60 chars, contains the primary keyword
- `<meta name="description">` — present, max 160 chars, unique across the site
- `<link rel="canonical">` — present, points to the correct URL
- `og:title`, `og:description`, `og:image` — all present and correct
- `<script type="application/ld+json">` — present and contains the correct schema type

### Step 3 — Validate JSON-LD

Paste the JSON-LD block into Google's Rich Results Test or Schema.org validator:

- No errors reported
- Schema type matches the page content (e.g. `Article` for blog posts, `BreadcrumbList` for inner pages)

### Step 4 — Check the sitemap

Open `/sitemap.xml` and confirm the page URL is present.
If the sitemap is generated by a Celery task, verify the task ran successfully after the page was published.

### Step 5 — Check robots.txt

Open `/robots.txt` and confirm the page path is not blocked by a `Disallow` rule.

### Step 6 — Run Lighthouse

Run a Lighthouse audit against the page in a production-like environment (staging or dev):

```text
Chrome DevTools → Lighthouse → Navigation → Desktop + Mobile → Analyse page load
```

Record the scores. All Core Web Vitals must meet the targets:

| Metric | Target   |
| ------ | -------- |
| LCP    | < 2.5 s  |
| CLS    | < 0.1    |
| INP    | < 200 ms |

Save the Lighthouse report (JSON or HTML export) to `project-management/src/11-SEO/`.

Name the file: `LIGHTHOUSE-[US###]-[ROUTE]-[DD-MM-YYYY].json`

### Step 7 — Check images

Inspect all images on the page:

- Every `<img>` has a non-empty `alt` attribute
- Decorative images use `alt=""`
- `alt` text is descriptive and contextually useful, not keyword-stuffed

### Step 8 — Verify heading hierarchy

Read the page source or use a browser extension (e.g. Headings Map) to confirm:

- Exactly one `<h1>` exists on the page
- `<h2>` and `<h3>` are used in logical order — no skipped levels

### Step 9 — Check the URL slug

Confirm the page URL is:

- Lowercase
- Hyphenated (no underscores or spaces)
- Human-readable
- Contains the primary target keyword

### Step 10 — Tick off the story SEO Acceptance Criteria

Open the story file and tick each item in `### SEO Acceptance Criteria`.
All items must be ticked before the story's Definition of Done can be marked complete.

### Step 11 — Commit any fixes

If any SEO issues were found and fixed during this workflow, commit the changes:

```text
/syntek-dev-suite:git
```
