# code/src/mobile/.storybook — Mobile Storybook Configuration

On-device Storybook configuration for `@storybook/react-native`. Stories render inside the app on a real device or simulator via Expo Go.

## Files

| File                       | Purpose                                                                  |
| -------------------------- | ------------------------------------------------------------------------ |
| `main.ts`                  | Story discovery globs and addon registration                             |
| `index.ts`                 | Exports `StorybookUIRoot` — consumed by `../app/_layout.tsx`             |
| `preview.tsx`              | Global decorators and parameters applied to every story                  |
| `../storybook.requires.ts` | Auto-generated story registry — regenerated by `pnpm storybook:generate` |

## Starting Storybook

```bash
# From the project root
bash code/src/scripts/development/server.sh storybook:mobile
```

This sets `EXPO_PUBLIC_STORYBOOK_ENABLED=true` and starts Expo. The root layout detects the flag and renders `StorybookUI` instead of the app. Scan the QR code with Expo Go to open Storybook on your device.

## Story discovery

`main.ts` discovers stories from two locations:

- `../src/**/*.stories.@(js|jsx|ts|tsx)` — mobile-specific component stories
- `../../shared/src/**/*.stories.@(js|jsx|ts|tsx)` — shared component stories

## Regenerating the requires file

After adding or removing story files, regenerate the registry:

```bash
bash code/src/scripts/development/server.sh storybook:generate:mobile
```

Commit the updated `storybook.requires.ts` — it must be present at build time.
