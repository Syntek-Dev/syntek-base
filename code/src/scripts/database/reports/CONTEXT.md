# code/src/scripts/database/reports

Output directory for backups and reports generated by the database scripts in `../`.

## Directory Tree

```text
code/src/scripts/database/reports/
├── CONTEXT.md               ← this file
├── .gitignore               ← ignores all generated files; keeps tracked files
├── .gitkeep                 ← keeps this folder present in git when empty
└── backup-<TIMESTAMP>.*     ← generated by backup.sh  [gitignored]
```

## Tracked Files

| File         | Purpose                                                                      |
| ------------ | ---------------------------------------------------------------------------- |
| `.gitignore` | Ignores generated output; keeps `.gitkeep`, itself, and `CONTEXT.md` tracked |
| `.gitkeep`   | Keeps this folder present in git when empty                                  |
| `CONTEXT.md` | This file                                                                    |

## Generated Files (gitignored)

| Filename pattern          | Source script                        |
| ------------------------- | ------------------------------------ |
| `backup-<TIMESTAMP>.dump` | `backup.sh` (custom format, default) |
| `backup-<TIMESTAMP>.sql`  | `backup.sh --format plain`           |

Backup files are never committed. Use `--output-dir` to write to a persistent location
outside the repo if you need to retain backups between cleans.
